# Owner 业务输入窗口

> Owner 业务输入是"业务真相"的源头。本文件规定**怎么收 / 怎么入册 / 怎么承载**——确保 Owner 任意时刻给的业务输入不流失、不变形、不延迟。
>
> 元约束：[守则 § 1 业务唯一裁判](../守则.md) + [W-协议](./W-协议.md)。

---

## § 1 三档分类

按"对架构 / 业务的影响半径"分三档：

| 档 | 范围 | 典型场景 | 入册位置 |
|---|---|---|---|
| **细节级** | 单 spec / 单页面 / 单字段 | 「这个按钮文案改成 X」「这个字段不要展示」 | 对应 spec / domain 行内 verbatim 摘录 + 改动落地 |
| **跨包契约级** | 多个 spec / 跨域接口 | 「分润机制实现时机」「派生树客户端可见度」 | DEVNOTES D 行（B 型）+ 涉及 spec 的 cross-ref 双向同步 |
| **设计岔路口** | 影响后续多轮决策的元方针 | 「方法论替代不了理解任务」「角色是理解方法的工具」「T3 不规范化」 | 守则（如属元原则）/ 经验库（如属方法）+ DEVNOTES D 行索引 + 整库引用更新 |

→ **判断难时按"取大"原则**：宁可走更高一档，避免 verbatim 流失。

---

## § 2 verbatim 入册纪律

Owner 业务输入的**原话**必须保留 verbatim，禁止以下操作：

- ❌ 摘要 / 提炼 / 改写为"AI 风格"
- ❌ 用「Owner 大意是 X」代替 Owner 原话
- ❌ 跨段拼接（A 段+B 段拼成一段）使 verbatim 失真
- ❌ 翻译为业务术语之外的工程术语

**正确做法**：

```
> Owner verbatim（YYYY-MM-DD / 第 N 次会话）：
> 「<原话整段照搬>」
```

> verbatim 摘录块必带：日期 + 会话编号 + 引号 + 整段。一句话 verbatim 也用引号。

## § 3 三处承载

Owner 业务输入按其性质，分三处长期承载：

| 承载点 | 内容性质 | 例 |
|---|---|---|
| **DEVNOTES § 二 决策简表 D-XX** | 业务规则 / 跨包契约 / 元方针 | D-64 头像授权双触发 / D-114 备份纪律 |
| **`doc/domain/` 各业务文件** | 业务概念明文化（D3 三谱系第一谱）| 帮找代币代收代缴规则 / 联合发布草稿四态 |
| **任务窗口 verify 段**（TP/SP/Step 文件内）| 任务专用、有时效的业务输入 | "本任务不考虑 X 场景"「本期只做自发模式」 |

> **不入册的下场**：业务输入在某次会话被"消化"但未入册 → 下一会话失忆 → 重新讨论或决策偏移。

## § 4 Phase-A 资料盘点入册流程（吸收原 Phase-A 子流程）

主题包 / TP 起步阶段的"资料盘点 Phase A"是 Owner 业务输入窗口的**集中触发场景**。流程：

1. **列 Q 项清单**：把所有需 Owner 拍板的问题成段写出（每题独立 Q-1 / Q-2 …）
2. **每 Q 项标谱系**：业务规则明文化 / 认知校准 / UI 显化（参见 [D3-三谱系.md](./D3-三谱系.md)）
3. **现场抛 Q 项**：W-重模式 / Owner 在场（参见 [W-协议.md § 5](./W-协议.md)）
4. **verbatim 收录**：Owner 每句拍板照搬入对应 Q 项答案段
5. **三处承载落位**：按 § 3 表分流入 DEVNOTES / domain / 任务窗口 verify
6. **关闭 Q 项**：所有 Q 关闭 → Phase A 结束 → 才可进入加工 Phase

## § 5 跨会话失忆兜底

Owner 业务输入可能在以下时机进入：

| 进入时机 | 兜底机制 |
|---|---|
| ① 上一会话讨论中 | 上次会话收尾时 verbatim 入册 + CONTEXT § 2 滚动留印 |
| ② 本会话进行中 | 当场 verbatim 摘录到任务窗口；会话结束按 § 3 三处承载分流 |
| ③ Owner 间歇说 `owner-inbox` | inbox 顶部追加 verbatim；下一会话开场必扫，按性质分流 |
| ④ Owner 主动改 doc/domain | 参谋长发现后立即按 § 3 同步至 DEVNOTES + 影响的 spec |

> 关键：**任何 Owner 业务输入都不能只活在"当前会话上下文里"**——必须在会话结束前落到某个长期承载点。
