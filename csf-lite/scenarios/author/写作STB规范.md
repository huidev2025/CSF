# 写作 STB 规范

> STB = Simple Task Brief。开发有开发的 STB 四件，写作有写作的 STB 七项——同一个概念，不同场景不同字段。
> 本文件定义写作场景的 STB 模版。写作者拿到 STB 后应能独立写作，不反复回头问。

---

## §0 为什么写作场景也需要 STB

开发场景的 STB（修改范围、参考范围、DT 清单、验收断言）解决了"代码改错地方代价大"的问题。

写作场景的 STB 解决另一个问题：**写作是非线性的。** 写作者今天写到一半，下次打开可能是三天后。没有 STB，写作者每次重启都要重新理解"我在写什么、写给谁、风格什么样"——这消耗的是参谋长的上下文管理带宽。

STB 的目的：**让参谋长把上下文一次性交给写作者，写作者拿到后独立工作。**

---

## §1 STB 字段

以下 7 项为写作 STB 的固定字段。第一项（目的与读者）为必填，其余按任务性质选填——长文多填，短文少填。

### ① 目的与读者（必填）

> 这篇东西要达到什么效果？谁在读？读者什么背景、什么期待？

```
目的：
读者：
```

**反例**："写一篇关于 CSF 的文档"——目的和读者都模糊，写了等于没写。
**正例**：目的：让首次接触 CSF 的用户 5 分钟内理解核心概念并知道怎么开始。读者：有编程背景、用过 VS Code、没接触过方法论框架的开发者。

### ② 梗概（选填）

> 叙事类写作（故事、案例、场景描述）用梗概。非叙事类（技术文档、报告、论述）可用"内容摘要"代替。

```
梗概：（两三句话的故事核心——谁、发生了什么、结果）
```

**适用场景**：小说章节、案例故事、场景还原、叙事性报告。技术文档可跳过此栏。

### ③ 写作大纲（选填）

> 文章的结构骨架。参谋长给层级和要点，写作者决定具体展开方式。

```
大纲：
- 开头：（要建立什么认知/引出什么问题）
- 主体①：（要点）
- 主体②：（要点）
- 结尾：（要留下什么）
```

**粒度**：大纲只到二级标题，不到段落。参谋长不代写——给方向，不给句子。
**长文必填**：超过 2000 字的任务，大纲必填。

### ④ 写作范围（选填）

> 写什么、不写什么。明确边界，防止写作者过度展开或遗漏。

```
包含：
不包含：
```

**示例**：包含：CSF 的三个核心概念（基线-log、业务唯一真理、目的替代记忆）。不包含：与其他方法论（GTD、Zettelkasten）的对比——不在本文范围。

### ⑤ 风格与语气（选填）

> 如果项目已有风格指南，指路径。如果没有，用关键词描述。

```
风格参照：（已有文本路径，或关键词描述）
语气：（正式/半正式/口语；客观/个人化；精炼/详尽）
篇幅：（大致字数范围）
```

**首次使用时必填**。后续同一项目可写"风格同上篇"。

### ⑥ 参考素材（选填）

> 写作者可以/必须参考的已有文件。

```
素材：
- （文件路径）：（与本次任务的关联）
```

### ⑦ 完成标准（选填）

> 怎么判"写好了"？可模糊，但要有。

```
完成标准：
```

**示例**：Owner 读完能用自己的话向同事解释 CSF 三个原则。或：初稿不追求完美，结构通顺、无事实错误即可。

## §2 STB 的使用

### 参谋长侧（写 STB）

- STB 在 `cos-context.md` §收件箱 中交付，格式 `<!-- @MSG:N -->...<!-- @/MSG:N -->`，内容即为按本规范的 7 项填写
- 如果写作者在上下文体系外（不读 cos-context.md），STB 通过 dev-context.md §收件箱 或直接文件交付
- **不要写到完美才派发。** 填你确定的，不确定的标 `[待确认]`，写作者看到会问或自主判断

### 写作者侧（接 STB）

- 拿到 STB 先扫 7 项。有标 `[待确认]` 的项 → 一次性问清，不逐条追问
- 7 项都清楚 → 开始写。中途发现矛盾 → 报告，不停下猜
- 完成后对照完成标准自检

### 验收

- 参谋长对照 STB 审稿。审稿意见逐条标注：对应 STB 第几项、是遗漏/偏离/质量问题
- 写作者逐条响应（接受修改 / 不接受+理由）
- Owner 最终确认

---

## §3 STB 的质量判断

好的 STB = 写作者读完后能回答以下问题（不回头问）：

- 我要写什么？（梗概+大纲）
- 写给谁？（目的与读者）
- 边界在哪？（写作范围）
- 什么语气？（风格与语气）
- 有什么可参考？（参考素材）
- 写到什么程度算完？（完成标准）

如果写作者读完后**问了一个 STB 应该覆盖的问题**——STB 写得不够好，参谋长补上。
