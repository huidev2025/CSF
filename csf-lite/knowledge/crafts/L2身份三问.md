# L2 身份三问（who-how-when）

## 工艺

选定 L2 身份后，必须附加三问，否则标签会滑向最易表述的低层动作：

| 问 | 内容 |
|---|---|
| **who** | 给谁用（受众：参谋长/Owner/开发者） |
| **how** | 怎么用（以什么专业眼光动手） |
| **when** | 什么场景/什么时机 |

## 为什么需要

session-159 实战暴露：贴了「业务领域分析师」身份标签后，AI 没有三问支撑，滑向了最易表述的机械动作——合规检查、行号精确化、同类项归集。真正的业务分析能力（识别遗漏规则 / 推导不变量 / 发现业务矛盾）被标签的空洞性淹没。

**身份标签不可独立使用** = 必附 who-how-when 三问。

## 应用

在 CONTEXT §2.0 下次启动指引中：
- 不只写「建议 L2 身份：业务领域分析师」
- 要写：who=参谋长自用+开发者参考 / how=从 domain 出发验证 Slice 规则完备性 / when=内容层复审阶段

## 自检

设定 L2 时问自己：如果只保留身份标签、删掉三问，我还知道具体怎么动手吗？如果不知道 → 三问没到位。
