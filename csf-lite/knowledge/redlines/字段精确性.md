# 字段精确性（候选标签即"字段"）

## 原则

在文档中引用标识符（编号、标签名、进度位、spec 名、状态值）时，必须逐项对账原始来源后引用。不得凭脑补补全描述性短语。

## 原因

session-136 元事故：收尾时引述下次会话候选标签 TD-01 / TC-XX，AI 凭印象补全了描述性短语（"TD-01 = 024 反向闯关插件"），实际 TD-01 = H5 共享组件库。导致 13 处错误标签扩散到多个文件。

**根因**：AI 把"标签"当作可以凭印象推导的"文字描述"，实际上标签是**精确字段**——引用必须回源校验。

## 防御方法

| 触发场景 | 纪律 |
|---|---|
| 引用 TD-XX / TC-XX / TA-XX 标签 | 1 行 doc/ 校验后引用 |
| 写进度位 N/24 → N+1/24 | 逐项对账主题分包清单 |
| 引用 spec 编号+描述 | 读 spec 文件确认标题 |
| 写状态变更（🔲→🟡→✅） | 读目标文件确认当前状态 |

## 自查口诀

命中以下任一 → 停下去读原始文件：
- "这个标签应该是…"
- "如果没记错是…"
- "它的描述大概是…"
