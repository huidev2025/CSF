# 凭印象推理 = 红线

## 原则

关键推理必须基于本会话真读的事实，不许凭印象。AI 没有可靠的跨会话记忆——"我记得"在这个系统中不构成证据。

## 原因

AI 的"印象"本质是训练权重 + 上下文残留，不是真实记忆。凭印象推理 = 在沙子上盖楼，错误率极高且难以自查（因为"感觉对"）。这是 v3 项目中复发最频繁的红线。

## 防御方法

| 防御面 | 做法 |
|---|---|
| 业务真相 | 不查 doc/domain 不推业务规则 |
| 文件内容 | 不读文件不引用其内容（read-before-write） |
| 跨会话状态 | "我曾经读过 X" = 幻觉触发口，必须重读 |
| 二手信息 | summary / sticky 不替代原文 |

**自查口诀**：命中以下任一 → 停下退回备料：
- "我曾经读过 X"
- "上下文里有 X 这个词"
- "凭印象应该是 Y"
- "sticky / summary 说过"
