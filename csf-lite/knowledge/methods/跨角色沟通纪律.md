# 跨角色沟通纪律（物理隔离 + Owner=提醒钟）

## 方法

多角色（参谋长 + 开发者）在 VSCode 环境中物理隔离：不同 chat 窗口、不能直接通话。跨角色信息传递只能通过文件。

## 三条铁律

1. **未落文件 = 对方不可见** — 口播/会话内对话/Owner 看到但未写入文件的 → 对方 AI 实例看不到
2. **Owner = 提醒钟，不是信使** — Owner 只说"对方有进展请查看对方 context"，不拷贝粘贴传话
3. **跨会话沟通必经文件** — CONTEXT.md / CONTEXT-开发者.md / PENDING / exec-log / 任务书

## Owner verbatim

「是向我确认，但并非由我拷贝粘贴来替你们传话。你们之间的沟通依然通过 pending、日志和其他文件。我只是负责提醒双方"有进展，请查看"。」

## 启动协议互读

- 参谋长启动：读自己 CONTEXT + **读开发者 CONTEXT 全文**
- 开发者启动：读自己 CONTEXT + **读参谋长 CONTEXT**
- 目的：了解对方最新进展 + 同步状态

## 适用场景

多 Agent 协作时（参谋长 + 开发者分工执行）。单 Agent 模式下本条目不激活。
