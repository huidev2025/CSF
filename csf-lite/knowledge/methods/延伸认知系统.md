# 延伸认知系统（Q3 自然法则）

## 方法

LLM 的物理事实直接驱动工作方法。理解物理约束 → 自然推导出需要什么职业能力。

## Q3 自然法则表

| LLM 物理事实 | 对应职业能力 |
|---|---|
| 上下文窗口有限 | 主动建延伸认知系统（笔记+索引） |
| 不能一次性查阅完所有相关资料 | 主动假设驱动探查 + 拼合结论 |
| 任何资料不在窗口内即不可见 | 主动留指针（自然语言锚点） |
| 推理可以并发但叙述串行 | 主动分主题收集 / 不混叙 |

## 核心推论

**延伸认知系统 = 一个任务**。准确描述：为了特定目的建立特定的信息索引系统，并为减少任务执行中的上下文空间需求做好准备。

这意味着：
- 笔记/索引/指针不是"附加的好习惯"，是 AI 的**基础生存能力**
- 建立索引本身值得投入设计精力（不是随手记）
- 索引范式 = 自然语言锚点（让下次读时激活理解，而非 RAG 式机械检索）

## 自然语言反 RAG 索引范式

传统 RAG：切块 → embedding → 向量检索 → 拼入 prompt
本范式：**用目的描述作索引** → 读时直接激活理解 → 不需要回放事件序列

好的索引条目写法：写目的和方向，不写流水账。让 AI 读到后直接进入工作状态。

## 应用

- CONTEXT §B 任务窗口的写法：目的叙述 > 事件记录
- 经验库条目的写法：原则 + 原因（激活理解）> 规则列表
- 会话笔记的结论段：用能激活理解的语言，不是信息罗列
