# E8 反馈通路（反馈素材池 → 升迁箭头）

> **CSF 的反馈机制**。E8 是 CSF 第 8 个要素 = 反馈素材池，但素材本身不是终点；终点是把素材**提炼为可复用条文**回流入 E2 守则 / E5 协作规范 / E6 方法论库，让 CSF 自身随经验进化。
>
> 本文件定义这条反馈通路的**五阶段标准流程** + **两层内化时序** + **灰度升级过渡**。

---

## § 1 五阶段标准流程

```
T0 收集 ─→ T1 落地 ─→ T2 跟踪 ─→ T3 入库 ─→ T4 提升
事件发生   即时落记    反馈计数    分类归档    内化为 CSF
                       决策升迁    案例 / 候选  框架部分
                                                ↓
                                  E2 守则 / E5 规范 / E6 工艺
```

| 阶段 | 动作 | 承载位 | 谁主导 | 时机 |
|---|---|---|---|---|
| **T0 收集** | 识别事件值得入素材池（事故 / 复盘 / 洞察 / 复发） | 会话内 / 测试现场 / 复盘时 | 参谋长 + 开发者 | 即时识别 |
| **T1 落地** | 落到素材池承载位 + 写清事件/触发面/当时处置；**若属 D 行复发 / 同时在 D-{N} 本体追加「复发记录区」一行**（修订 1 / 详 § 1.1）| session-NNN.md 笔记 / DEVNOTES / §B 备忘区 / TP 任务文件夹 / 经验库 + D-{N} 本体 | 识别者 | **即时**（不等收尾 / 202+204+205+208 范式）|
| **T2 跟踪** | 复发时追加复发记录 + 计数 + 触发面归类 | 原承载位（追加在原条下 / 不新立条） | 识别者 | 复发当时 |
| **T3 入库** | 阈值触发 → 分类归 notes/教训库 子目录；**同主题谱系合归 `notes/教训库/案例-按主题/D-{N}-谱系/`**（修订 3）| `notes/教训库/{案例-按主题/D-{N}-谱系/, 升迁候选/, 已升迁存根/}` | 参谋长 | 提炼回合 |
| **T4 提升** | 升 E2 守则 / E5 协作规范 / E6 方法论库 + 案例归档 + 原素材池条目标记「已升迁」 | E2/E5/E6 + `notes/教训库/已升迁存根/` | 参谋长 + Owner 拍 | 提炼回合 |

### § 1.1 D 行立条规约（修订 1+2 / 209-试点 § 4）

**立独立 D 行文件时**（`notes/devnotes/D-{N}.md`）强制含 **§ 复发记录区**（即使建立时为空）：

```markdown
## § 复发记录

| # | 会话 | 触发面 | 落地承载位 |
|---|---|---|---|
| ① | （立条时为空 / 复发时 append）| | |
```

**T1 落地双写**：D 行类素材复发时 / T1 即时落库的承载位 ≠ 替代 D 本体 / **必须双写**：
1. 当次承载位（session-NNN.md 笔记 / TP 任务文件夹 / §B 备忘区）= 写完整事件
2. D-{N} 本体「复发记录区」append 一行（会话号 + 触发面 + 指针 → 当次承载位）

**理由**：D 本体作为复发汇总单一信源 / 否则参谋长无法基于「复发 ≥ 3 次」触发 T3/T4 仪式（209-试点 § 3.3 验证缺失后果）。

### § 1.5 非升迁合法形态三类（修订 5+6+7 / 209-c § 4）

**关键认知**：E8 通路 ≠「所有素材都必须走 T4」。以下三类**合法非升迁**，不视为通路失败 / 但需明示停留状态与理由。

| 类型 | 含义 | 案例 | 标识方式 |
|---|---|---|---|
| **a. 设计不升（边界类）** | 显影类 / 灵感库类 / 阶段性留痕类产物 / 主动停在 T1 不消化 | tp-架构形式化 阶段 1（找感觉 / 不立条款）/ tp-业务切片 R1.x（随归档消失） | TP README 明示「不立条款」+ 设计意图 verbatim 留痕 |
| **b. 业务产物豁免** | 业务交付物本身（非协作机制类） / 不属升迁范畴 | Slice 10 篇 / 4 接面契约 / GP 池表 / 业务索引 | 直接 T3 业务交付 / 不参与升迁判级 |
| **c. 资产复用类** | 跨项目级可复用资产（非规范条文）/ 不升 E2/E5/E6 但应登记 | TP-bc 双角色空模板 + Lite / 跨项目工具模板 | 加入「跨项目复用资产登记表」（INDEX § 0.X 候选 / 或 E6 方法论库登记）|
| **d. 下游纯引用层**（213 立 / TP-T4 §7.4）| 下游本体引上游元原则 / 不重复提炼 / 已被既有 D 行或既有节覆盖 | TP-T4 §1 / §1.2 / §2 三段引 D-127 / D-129 / D-132 等上游条文 = 不另立条 | 评估表标「已等价覆盖」+ 列出被引上游条 |

**判定指引**：
- 是规范/纪律/工艺？→ 走 T4 升 E2/E5/E6
- 是业务交付物本身？→ 类型 b / N/A
- 是显影/灵感/找感觉？→ 类型 a / 明示停 T1
- 是跨项目可复用模板/工具？→ 类型 c / 登记不升迁
- 是引用上游已立条款？→ 类型 d / 已等价覆盖不重复

### § 1.5.1 单层升 vs 多层升（213 立 / TP-T4 §7.3）

**v0.2 隐含二分**（升 E2 或 升 E5）/ TP-T4 实战发现 **2 项「两升」** = 同一条同时落 E2 元原则 + E5 接口规则两层。判定矩阵扩为三列：

| 列 | 含义 | 触发 | 例（TP-T4 实战）|
|---|---|---|---|
| **单层 · E2** | 仅升元原则 / 不需接口侧执行规则 | 是裁判尺度类 / 不需开发侧执行步骤 | FLDD §3 本体（精准修改的元原则）|
| **单层 · E5** | 仅升接口规则 / 元原则已被既有条覆盖 | 是执行细节类 / 不构成新元原则 | FLDD §3.1 范围契约清单格式 |
| **双层 · E2+E5** | 元原则 + 接口执行规则同时升 | 既有裁判尺度（薄壳）也有执行步骤（详则）/ 二者对应不同读者 | FLDD §4 验收三档 + §5 沉淀级别（两项均「两升」）|

**判定要点**：
- 「能在 E2 用 ≤1 段写完且开发者无需细化执行」→ 单层 E2
- 「只有执行细节、无新裁判尺度」→ 单层 E5
- 「元原则与执行细节都不可省」→ 双层（E2 薄壳引 E5 详则 + E5 反向锚 E2）

### § 1.6 提炼回合候选状态跟踪表（213 立 / TP-T4 §7.1）

**背景**：T2 跟踪默认人工复查 / 一次查一批不记「多候选并行状态」。209-c 识别 8 项提炼回合 #2 候选 / 仅 A 项走 T4 / B-H 进入未提炼状态后无位记录。

**表格模板**（参谋长维护 / 推荐放在提炼回合 TP 根目录或 CONTEXT 附录）：

| # | 候选名 | 识别会话 | 当前状态 | 下会话动作 |
|---|---|---|---|---|
| 1 | （例）FLDD 5 条 | 209-c | c. 已提炼（TP-T4 / 212···完成）| — |
| 2 | （例）CONTEXT 三门槛 | 207 | c. 已提炼（213 升 E2 § 12）| — |
| 3 | （例）待办流改造 | 207 | a. 待入提炼 | 214 启动 |

**状态四态**：
- **a. 待入提炼** = 已识别、未起 TP
- **b. 提炼中** = TP 已立 / T4 未完
- **c. 已提炼** = T4 已落 / D 行已立
- **d. 取消** = 后续评估为不需升迁 / 或被别项覆盖

**使用**：T3 入库或 T4 完成后起手更新。未创建此表之前候选仅依赖会话笔记 + sticky / 不可复查。

## § 2 两层内化时序（关键设计 / 防误读）

E8 通路的内化效力**不是只在 T4 才产生**。两层时序：

```
T1 即时内化   ──→  当前项目即生效
              （下次会话起不再犯 / 落库 = 强执行）
                            ↓
T4 完整内化   ──→  新项目直接可用
              （条文升 CSF / 跨项目复用）
```

| 层 | 时点 | 效力范围 | 机制 |
|---|---|---|---|
| **T1 即时内化** | 落库瞬间 | 当前项目（本会话 + 后续会话）| 写进 context §B 备忘区 / DEVNOTES / 经验库 = 引擎“写到基线就强执行”机制 |
| **T4 完整内化** | 提炼回合后 | 跨项目（CSF 框架级 / 新项目继承）| 升 E2/E5/E6 条文化 / 与项目实例解耦 |

**为什么必须分两层明示**：
- 误读 1（已发生）：等 T4 才有用 → 落库懒得做 → 当前项目继续踩坑（违反即时落库范式）
- 误读 2：T1 落库 = 任务完成 → 永远停在素材池 → CSF 不进化 → 新项目从零开始踩同样坑
- 防御：本文件明示「T1 已生效 + T4 是另一层升级 / 不互斥 / 不可省」

## § 3 提炼回合（T3+T4 仪式）

**定义**：把素材池条目按触发条件批量过 T3 入库 + T4 提升的集中动作。**不是常规收尾动作**，是阈值/召回/收口触发的独立仪式。

**触发条件**（任一满足）：

| 触发源 | 条件 | 主审 |
|---|---|---|
| **参谋长主动** | SP 收口 / SEC-2.0-D > 5 条触发外迁 / 某条 D 复发 ≥ 3 次 / 单批 DEVNOTES > 3 新 D / 单类教训积累 ≥ 5 条 | 参谋长 |
| **Owner 召回** | Owner verbatim 召回（如 [201 inbox 🟢 设想](../owner-inbox.md)）| Owner + 参谋长 |
| **阶段切换** | GP 阶段切换 / SP 结案 / Phase 切换 | 参谋长 |

**仪式步骤**（参谋长执行 / Owner 校验 T4 升迁动作）：

1. **盘点**：列素材池所有条目（§B 备忘区 + DEVNOTES + 经验库/ + TP 任务文件夹内笔记）/ **同时识别「半 T4 状态」项**（详 § 3.1 / 修订 4）
2. **分类**：按触发面归类（红线类 / 工艺类 / 沟通类 / 业务认知类 / ...）/ **同时标记非升迁三类**（§ 1.5 a/b/c）→ 类型 a/b/c 跳过判级
3. **判级**：每条标 T3 还是 T4
   - **仅 T3**（保留为案例 / 不升条文）：单次性 / 触发面太窄 / 不可复用
   - **T4 候选**（升 E2/E5/E6）：复发 ≥ 2 / 触发面通用 / 可条文化
4. **写候选稿**：T4 候选条目写升迁稿（目标条文 + 原素材引用 + 影响面）
5. **Owner 拍 T4**：参谋长出 L3 brief 列升迁候选 + Owner 拍 ACCEPT/MODIFY/REJECT
6. **落地**：
   - T3 入库：归 `notes/教训库/案例-按主题/D-{N}-谱系/`（同主题样本聚合 / 修订 3）或 `notes/教训库/升迁候选/` 对应子目录
   - T4 提升：改 E2/E5/E6 文件 + 原素材池条目末尾追加「✅ 已升迁至 [文件 § X]（YYYY-MM-DD）」+ 原始事件归档 `notes/教训库/已升迁存根/`
7. **存根**：素材池原条目**保留**（不删 / 便于回溯 / 末尾标已升迁状态）

### § 3.1 半 T4 状态识别（修订 4 / 209-试点 § 3.5）

**定义**：某素材已**部分自发完成 T4**（个别条文已升 E2/E5/E6）但**未走完整提炼回合仪式**（无 L3 brief + Owner 拍 ACCEPT 显式动作 / 复发后未触发第二轮升迁）。

**典型样板**：D-130 凭印象=红线
- ✅ 守则 § 3.7 + Slice § 2.0.0 已条文化（部分 T4）
- ❌ 后续 6+ 次复发未触发第二轮 T4（已超阈值未补走）

**识别方法**（盘点时三问）：
1. 该素材是否已有条文出现在 E2/E5/E6？
2. 该条文化是否经过显式 Owner 拍板 + L3 brief？
3. 后续复发是否补走过提炼回合？

三问任一为 No / 但已部分升 = 半 T4 → 应纳入下一次提炼回合补走仪式。

**防御目的**：防止「差不多就行」漏判 / 部分 T4 容易被误认为「已完成升迁」从而错失新一轮提炼。

## § 4 灰度升级过渡（历史包袱处置）

**原则**：通路从 209 起生效。**不强行追溯改写历史**。

| 时段 | 处置 |
|---|---|
| **209 之前** | 历史素材保持原样（已外迁的 169-179 归档不动 / SEC-2.0-D 现存 50+ 条不强制重排）|
| **209 起** | 新增事件按本通路走（T0 即识 → T1 即落 → T2 即跟 → T3/T4 仪式批量）|
| **存量消化** | 独立多会话渐进做（参谋长按阈值主动发起 / 不挤占常规会话 / 不在本会话 209 内做）|

**存量消化触发**：
- 当某类历史素材积累 ≥ 5 条且参谋长判定"现在批量提炼成本低收益高" → 起独立 TP / 走完整提炼回合
- 候选优先级（参谋长后续判定）：（a）凭印象=红线 D-130 谱系 ≥ 6 次复发（本会话试点）（b）multi_replace 工艺谱系 ≥ 5 次（c）assign 双写补丁谱系 ≥ 3 次 ...
