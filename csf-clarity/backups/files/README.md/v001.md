# CSF Lite

> **版本**：v1.0
> **用途**：CSF 方法论最小可行产品。拷贝即部署，AI 读取后进入参谋长角色。
> © 2025 zhanghui. CC BY-NC 4.0. https://github.com/huidev2025/CSF

---

## 这是什么

CSF Lite 是一个文件夹。将它拷贝到你的项目工作区，AI 读取其中方法论文本后，即进入参谋长角色，与你展开协作。

---

## 目录结构

```
csf-lite/
├─ cos-context.md           ← 参谋长引擎（核心，每次会话必读）
├─ dev-context.md           ← 开发者引擎（软件项目用）
├─ QUICKSTART.md            ← ⭐ 首次使用从这里读
├─ README.md                ← 本文件
├─ clarity.exe              ← 脚手架（文件编辑 + 提示词面板）
│
├─ core/                    ← CSF 方法论内核
│   ├─ 参谋长-instructions.md
│   ├─ 参谋长-方法论.md
│   ├─ 开发者-instructions.md
│   ├─ 守则.md
│   └─ 协作规范.md
│
├─ protocols/               ← 操作规程（16个，按需加载）
│   ├─ FLDD-开发执行.md
│   ├─ 立项协议.md
│   ├─ BugFix-修复机制.md
│   └─ ...
│
├─ knowledge/               ← 可检索经验（methods/redlines/crafts）
│   ├─ methods/
│   ├─ redlines/
│   └─ crafts/
│
└─ workspace/               ← 项目运行文件
    ├─ DEVNOTES.md
    ├─ bug-tracker.md
    └─ owner-inbox.md
```

---

## 快速开始

1. 复制本目录到你的项目工作区根目录
2. 打开 [QUICKSTART.md](QUICKSTART.md) — 按 §二完成 Owner 填写
3. 告诉参谋长「阅读 cos-context」

---

## 文件修改说明

复制后，以下文件**需要填写项目信息**（含 `<!-- -->` 占位符）：

| 文件 | 需要填写的位置 |
|---|---|
| `cos-context.md` | §A > 项目与团队（项目名 + 描述）|
| `cos-context.md` | §B > 全景图、三元组指针（首次会话由参谋长填）；三元组文件在 `triplets/` 目录 |
| `dev-context.md` | §A > 信息可见范围（代码/日志目录路径）|
| `dev-context.md` | §B > 定位、进度表（由参谋长维护）|

以下文件**无需修改**，直接使用：
- `core/` 全部文件（参谋长-instructions.md、参谋长-方法论.md、守则.md、协作规范.md、开发者-instructions.md）
- `protocols/` 全部文件
- `knowledge/` 全部文件

---

## 注意事项

- **protocols/ 不需要预读**：按 `cos-context.md §A > 触发索引` 按场景加载
- **knowledge/ 不需要预读**：同样按触发索引按需调取
- **§A 不要随意改**：`cos-context.md §A` 是引擎核心，变更走 `core/守则.md §5`
- **FLDD 规程可选**：非软件开发项目可以不挂载 `protocols/FLDD-开发执行.md`
