# 上下文预算管理（信息分包两层结构）

## 方法

将 AI 需要的信息分为两层，严格控制各层体量，确保工作空间充足：

| 层 | 内容 | 预算 |
|---|---|---|
| **全局共识层** | 架构骨架 + 契约边界 + 设计哲学 + 核心代码示例 + 命名约定 | 30K（16%） |
| **主题专属层** | 当前任务的最小完备资料（允许跨包重复） | 50K（26%） |
| **工作空间** | 编码展开 + 多轮 Q&A + 工具反馈 + 安全余量 | 100K+12K（58%） |

**单包总上限 80K**（共识+专属），超限强制再切。

## 为什么有效

解决的核心矛盾：AI 需要全局视野（避免脑补/偏离架构），但上下文窗口有限。

- 不分包 → 全塞进去 → 超限 / 工作空间被压缩
- 分包不设预算 → 各包体量失控 → 某包超 50K 时工作空间不足
- 本方法 → 两层分离 + 固定预算 → 全局在场 + 工作空间充足

## 完备性自检三问

| 维度 | 自检问 |
|---|---|
| **自洽性** | 开发者仅凭此包能否独立完成？ |
| **封闭性** | 是否引用了包外未提供的资料？ |
| **精简性** | 是否包含与当前任务无关的内容？ |

## 适用场景

需要给开发者（或下一个 AI 实例）准备工作资料时——如任务书、主题包、STB。核心思想也适用于 CONTEXT 设计：精练的入口（全局共识）+ 按需加载的详情（专属层）。
