# STB API 引用须附源码行号

## 原则

STB 中每引用一个函数签名、类字段、方法名，**必须附上该签名在源码中的实际行号**。行号是"我确实读过这份代码"的物理证据——不是凭印象、不是凭设计理想。

## 原因

参谋长写 STB 时的危险模式：沉浸在体系设计中（擅长项），按"应该是什么样的 API"来写，而非按"实际是什么样的 API"来写。结果 STB 中的 API 描述是虚构的理想版本，开发者在三问承接时发现全部不一致。

**仅凭写前真读原则不够**——规则在但可能不被执行。行号是可验证的门槛：没行号 = 没真读。

## 操作手法

STB 中 API 引用格式：

```markdown
| `get_items() → list[Item]` | service.py:L379 | 解析数据返回条目列表 |
| `update_field(name, val) → None` | service.py:L496 | 更新指定字段值 |
| `Item.desc: Optional[str]` | models.py:L252 | 描述字段 |
```

**检查单**（STB 出稿前）：
- [ ] STB 中每个 API 引用旁有 `文件:行号`
- [ ] 行号来自本会话最近一次真读（非跨会话记忆）
- [ ] 签名与实际一致（参数名、返回类型、字段名）
- [ ] 如有不确定 → 标注 `<!-- TODO: 真读确认 -->`，写前补读
