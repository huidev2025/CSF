# 即时落库（有产出就落盘）

## 工艺

会话中产生阶段性明确产出时，**立即写入文件**。不等 88 收尾批处理。

## Owner verbatim

「首先把所有能落地的都落地，以免之后会话过长导致遗忘。甚至包括 context，如果有了阶段性明确产出，也没有必要等到最终收尾时才落地。」

## 为什么有效

1. **防遗忘**：会话越长，前面的产出越容易被后面的讨论冲掉
2. **防丢失**：万一会话中断（超时/崩溃），已落盘的不会丢
3. **减负收尾**：收尾时只处理最后一段产出，不用回溯全程
4. **即时可验证**：落盘后可以 read_file 确认正确性

## 触发时机

| 产出类型 | 落盘动作 |
|---|---|
| 新条目/新文件设计完成 | 立即 create_file |
| CONTEXT 某节有确定更新 | 立即 replace |
| D 行/决策产生 | 立即写入 DEVNOTES |
| 经验/教训识别 | 立即写入经验库 |

## 与收尾的关系

即时落库 ≠ 不需要收尾。收尾仍然做：
- 覆写 §C/§D 基线（只有会话结束才知道最终状态）
- 检查笔记完整性
- changelog 汇总

但收尾的工作量从"处理全程产出"变为"处理最后增量"。
