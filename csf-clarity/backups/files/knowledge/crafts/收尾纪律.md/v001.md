# 收尾纪律

> ⚠️ **已废弃**。本文描述的是旧系统收尾流程（§3/§4 覆写），已被 context §A 收尾协议替代。保留供历史参考。

## 方法

收尾 = 更新基线 + 确认日志完整。具体动作序列：

1. 回顾笔记 — 确认所有回合完整
2. 提炼元认知 — 有教训写入日志段
3. 覆写 context §3 — 更新任务窗口 = 新基线
4. 更新 §4 — 按需增删活跃防御
5. changelog — 在笔记末尾记文件变动

## 原因

收尾的本质 = 基线-log 分离的执行时刻。覆写 §3 = 旧状态自然沉入 log（存在于笔记中），基线只保留最新真相。如果收尾不做，下个会话读到的是过期基线。

## 关键约束

- AI 不可自动启动收尾（只能提醒）
- Owner 说"收尾"或"88" → 立即执行
- 上下文空间风险 → AI 预估提醒，不自动启动
