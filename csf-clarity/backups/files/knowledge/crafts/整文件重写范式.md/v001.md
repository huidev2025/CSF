# 整文件重写范式（大幅改动时）

## 工艺

当文件需要大幅改动（重构/升版/体积缩减 > 30%）时，不走 replace 增量补丁，而是：

```
Remove-Item → create_file 整文件从头写
```

一次成型，避免补丁堆叠。

## 为什么有效

session-216 实战：参谋长 instructions 308→120 行（-61%），从头重写比逐段 replace 更快、更准、更干净。

**对比两种路径**：

| 增量补丁路径 | 整文件重写路径 |
|---|---|
| 多次 replace → 每次都可能失败 | 一次 create_file → 必定成功 |
| 旧结构残留 → 越改越乱 | 从新结构出发 → 一次到位 |
| 需要逐行对比旧版 → 依赖印象 | 凭新设计 from scratch → 不依赖印象 |
| 适合 < 5 行小修 | 适合大幅重构/升版 |

## 适用判据

- 改动量 > 文件 30%
- 结构性重组（不只是内容更新，是章节重排）
- 升版（v0.1→v0.2 级别改动）
- 多次 replace 已失败或预计会反复失败

## 操作步骤

1. 设计好新文件的完整结构
2. PowerShell `Remove-Item -Path "..." -Force`
3. `create_file` 写入完整新内容
4. PowerShell `(Get-Content -Path "...").Count` 验证行数

## 不适用

- < 5 行小修 → 直接 replace_string_in_file
- 文件有其他人并发编辑风险 → 不能 Remove（会丢失他人改动）
