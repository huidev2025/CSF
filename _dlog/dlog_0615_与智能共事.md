Middle\"——上下文越长，注意力越涣散。\n\n**CSF v3 的解法**：在文件系统层面实现**冷热分离**——\n\n- **基线层**（`context.md` 等，**覆写式**）：永远只保留最新的当前状态。\n- **Log 层**（`session-NNN.md`，**追加式**）：保留完整讨论过程，需要时回溯。\n\n每次新会话启动，AI 读到的都是\"脱水\"后的高浓度有用信息，从根上规避了大模型注意力涣散的工程缺陷。\n\n#### ③ 多角色物理隔离 + 异步文件通信（Industrial-Grade Multi-Agent）\n\n**行业现状**：CrewAI / AutoGen 这类多 Agent 框架让 AI 之间互相对话——结果是\"无限套娃讨论\"\"无效信息轰炸\"\"幻觉互相加强\"。\n\n**CSF v3 的解法**：参谋长（设计态）和开发者（执行态）**完全不直接对话**。\n\n- 参谋长把设计转化为 **STB（任务书）**，开发者只读 STB，**禁止**读上游 domain/arch；\n- 开发者用**三栏自检**（✅已读懂 / 🟡不确定 / 🔵准备假设）标记理解状态，任一非空 → PENDING 回参谋长，绝不动键盘；\n- 通信通过文件落盘（`@MSG:N` The cognitive load on the human is enormous.\n\n**CSF v3**: the AI **runs itself end-to-end**. The \"engine\" section in `context.md` defines a strict opening protocol (L1 load -> L2 task-level alignment -> L3 concrete plan). After reading `context.md` the AI knows which chains to load, where to retrieve resources, when to log to `session-NNN.md`, when to recalibrate, and how to close out. Control of \"how to collaborate with the human\" is **handed off from human cognition to the AI's own self-procedure**. The human becomes a *commander* (confirm / correct) rather than a *scheduler*.\n\n#### ② Hot/Cold Separation in the Filesystem (Baseline–Log)\n\n**Industry today**: in a typical LLM session, fresh and stale information sit side by side in one long transcript — triggering the well-known \"Lost in the Middle\" attention degradation.\n\n**CSF v3**: cold/hot separation **at the filesystem level** —\n\n- **Baseline layer** (`context.md`, *overwrite-style*) keeps only the current dehydrated state.\n- **Log layer** (`session-NNN.md`, *append-only*) preserves the full reasoning trail for back-tracing when needed.\n\nEvery new session starts from concentrated, current information. The attention-degradation flaw is engineered around, not fought.\n\n#### ③ Industrial-Grade Multi-Role: Physical Isolation + Async File-Based Comms\n\n**Industry today**: multi-agent frameworks (CrewAI, AutoGen, …) let agents chat directly with each other. The common failure modes are \"infinite recursion of debate\", \"noise amplification\", and \"mutual hallucination reinforcement\".\n\n**CSF v3**: the Chief-of-Staff (design) and the Developer (execution) **never converse directly**.\n\n- The Chief-of-Staff writes a **STB (Simple Task Brief)** as the contract. The Developer is **forbidden** to read upstream `domain/` or `arch/`.\n- The Developer self-checks every input through a **three-column classifier** — ✅ understood / 🟡 unsure / 🔵 working assumption. If any non-✅ remains, the work is **PENDING'd back**; no keystrokes.\n- All cross-role communication happens through the filesystem with a sentinel format (`@MSG:N`), never as conversation.\n\n“Isolation + deterministic feedback” is mature industrial software engineering applied to the AI-collaboration setting.
